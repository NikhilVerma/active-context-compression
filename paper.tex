\title{Active Context Compression: Enabling ``Immortal'' Agents via Episodic Memory Consolidation}

\author{\IEEEauthorblockN{Nikhil Verma}

\begin{abstract}
Large Language Model (LLM) agents struggle with long-horizon software engineering tasks due to ``Context Bloat.'' As interaction history grows, computational costs explode, latency increases, and reasoning capabilities degrade due to distraction by irrelevant past errors. Existing solutions often rely on passive, external summarization mechanisms that the agent cannot control. This paper proposes \textbf{Focus}, an agent-centric architecture inspired by the biological exploration strategies of \textit{Physarum polycephalum} (slime mold). The Focus Agent actively explores a solution path, consolidates key learnings into a persistent ``Knowledge'' block upon reaching a milestone or dead end, and actively withdraws (prunes) the raw interaction history. Evaluated on the SWE-bench Lite benchmark, Focus demonstrates two key benefits: (1) for capable models like Claude Sonnet, it reduces token consumption by \textbf{51\%} while maintaining 100\% task success rate; (2) for smaller models like Claude Haiku, it \textbf{improves success rate by 50\%} (from 20\% to 30\%) by maintaining cognitive hygiene. We demonstrate that active context management converts the hard limit of context windows into a sustainable ``cognitive tax,'' enabling both cost savings and improved reasoning for resource-constrained models.
\end{abstract}

\section{Introduction}
The ``Context Window'' remains the fundamental bottleneck for autonomous AI agents. While recent models offer massive context capacities (e.g., 200k+ tokens), utilizing this capacity naively presents three critical challenges:
\begin{enumerate}
    \item \textbf{Cost:} In an iterative loop, re-processing a growing history for every new inference step leads to quadratic cost accumulation.
    \item \textbf{Latency:} Time-to-first-token degrades linearly with context length, making interactive agents sluggish.
    \item \textbf{Context Poisoning:} Long contexts filled with trial-and-error logs, failed tests, and verbose tool outputs can distract the model, leading to the ``Lost in the Middle'' phenomenon~\cite{liu2023lost}.
\end{enumerate}

\section{Related Work}

\textbf{Memory-Augmented Agents.} Several approaches address context limitations through external memory. MemGPT~\cite{packer2023memgpt} implements a virtual memory hierarchy inspired by operating systems, allowing LLMs to manage their own memory through explicit read/write operations. Voyager~\cite{wang2023voyager} uses a skill library to store reusable code snippets, reducing redundant exploration.

\textbf{Reflection and Self-Improvement.} Reflexion~\cite{shinn2023reflexion} enables agents to learn from failures by generating textual reflections stored in an episodic memory buffer. However, this operates \textit{between} task attempts rather than within a single continuous trajectory. LATS~\cite{zhou2023lats} combines tree search with LLM reasoning but still accumulates full trajectories.

\textbf{Context Compression.} StreamingLLM~\cite{xiao2023streamingllm} addresses infinite context through attention sink tokens, but operates at the attention level rather than providing agent-level memory management. LLMLingua~\cite{jiang2023llmlingua} compresses prompts but requires a separate compression model.

\textbf{Our Contribution.} Unlike prior work, Focus enables \textit{intra-trajectory} compression---the agent actively prunes its own history \textit{during} a single task, preserving learnings in a structured knowledge block. This complements rather than replaces external memory approaches.

Standard agents operate in an ``Append-Only'' mode: every thought, tool call, and system response is permanently added to the conversation history. This strategy works for short tasks but ensures failure for complex, open-ended exploration, as the agent inevitably hits the token limit or becomes confused by its own previous failures.

We propose a shift from \textit{Passive Retention} to \textit{Active Compression}. Inspired by the efficiency of \textit{Physarum polycephalum}---a slime mold that explores environments and physically retracts from dead ends while leaving chemical markers to avoid re-exploration---we introduce the \textbf{Focus Agent}. This agent is equipped with the ability to introspect, summarize its recent trajectory into a high-level learning, and physically delete the raw logs of that trajectory from its immediate context.

\section{Methodology}

\subsection{The Slime Mold Analogy}
Biological systems do not retain a perfect record of every muscle movement used to navigate a maze; they retain the \textit{learned map}. Similarly, an agent exploring a codebase does not need to remember the 50 lines of \texttt{ls -R} output from ten minutes ago; it only needs to remember that ``the config file is not in the \texttt{/src} directory.''

\subsection{Architecture: The Focus Loop}
The Focus architecture introduces two primitives to the standard ReAct agent loop: \texttt{start\_focus} and \texttt{complete\_focus}.

\begin{enumerate}
    \item \textbf{Start Focus:} The agent declares a distinct line of investigation (e.g., ``Debug the database connection''). This marks a checkpoint in the conversation history.
    \item \textbf{Explore:} The agent uses standard tools (read, edit, run) to perform work.
    \item \textbf{Consolidate (The Nudge):} When the agent completes the sub-task or hits a dead end, it is prompted to call \texttt{complete\_focus}. Crucially, the agent \textit{itself} generates a summary of:
    \begin{itemize}
        \item What was attempted?
        \item What was learned (facts, file paths, bugs)?
        \item What is the outcome?
    \end{itemize}
    \item \textbf{Withdraw:} The system takes this summary, appends it to a persistent ``Knowledge'' block at the top of the context, and \textit{deletes} all messages between the checkpoint and the current step.
\end{enumerate}

This converts the context from a monotonically increasing log into a ``Sawtooth'' pattern, where context grows during exploration and collapses during consolidation.

\section{Experiments}
We evaluated the Focus architecture on \textbf{SWE-bench Lite}, a benchmark for software engineering agents solving real GitHub issues. All evaluations used the official SWE-bench Docker-based harness to ensure accurate pass/fail determination. We ran experiments using two models: \texttt{claude-sonnet-4-5-20250929} (N=3) and \texttt{claude-haiku-4-5-20251001} (N=10) to test Focus across different model capability levels.

\subsection{Key Findings}

\textbf{Finding 1: Efficiency for Capable Models.} On Claude Sonnet (Table~\ref{tab1}), both agents achieved 100\% success rate on the tested Django instances. However, Focus accomplished this while consuming \textbf{51\% fewer tokens} and maintaining an 84\% smaller context. This demonstrates that context compression preserves problem-solving capability while dramatically reducing costs.

\textbf{Finding 2: Improved Success for Smaller Models.} On Claude Haiku (Table~\ref{tab2}), Focus achieved a \textbf{50\% higher success rate} than baseline (30\% vs 20\%). Notably, Focus solved \texttt{django\_\_django-11039} that Baseline could not. This suggests that maintaining ``cognitive hygiene''---keeping context clean and focused---helps smaller models reason more effectively by avoiding distraction from accumulated noise.

\subsection{Case Study: The Sawtooth Pattern}
The efficiency gain stems from the ``Sawtooth'' context pattern. In our Sonnet experiments, the Baseline agent's context grew monotonically to over 120 messages per task on average. In contrast, the Focus agent compressed its history multiple times (averaging 5 compressions per task), dropping raw logs while retaining high-level knowledge, finishing with lean contexts under 20 messages. This pattern enables the Focus agent to explore more deeply while maintaining clarity.

\subsection{Case Study: Cognitive Hygiene Helps Smaller Models}
The most striking result is Focus solving \texttt{django\_\_django-11039} with Haiku where Baseline failed. Analysis of the trajectories suggests that Baseline's context became polluted with verbose error traces and failed attempts, causing the model to lose track of the core problem. Focus, by compressing this noise into structured learnings, maintained clarity and ultimately found the solution. This supports the hypothesis that smaller models benefit disproportionately from cognitive hygiene.

\subsection{Analysis}
\textbf{Efficiency:} The Focus Agent reduced token consumption by 51\% on Sonnet while maintaining 100\% success. This reduction is primarily due to avoiding the quadratic cost of re-processing long histories. 

\textbf{Cognitive Hygiene:} By maintaining a context size under 20 messages on average (vs. 100+ for the baseline), the Focus agent mitigates the ``Lost in the Middle'' phenomenon, ensuring that instructions remain salient even after extended exploration.

\textbf{Improved Reasoning for Smaller Models:} Most significantly, Focus \textit{improved} success rates for Haiku by 50\%. This suggests that cognitive hygiene---keeping context clean and focused---is particularly valuable for less capable models that are more susceptible to distraction from accumulated noise. The compression mechanism not only preserves information but actively improves the model's ability to reason by reducing interference.

\subsection{Ablation: Compression Frequency}
We evaluate three compression strategies to understand the optimal balance between context retention and reduction:

\begin{itemize}
    \item \textbf{Fixed (15 steps)}: Aggressive compression to keep context lean.
    \item \textbf{Fixed (30 steps)}: Relaxed compression to allow more local context.
    \item \textbf{Model Triggered}: A fully autonomous mode where the agent decides when to consolidate context based on task completion, without external heuristics.
\end{itemize}

The optimal frequency likely depends on task complexity and the model's ability to self-regulate.

\section{Discussion}
\subsection{The Cost of Immortality}
Active compression introduces a ``cognitive tax''---the tokens spent generating summaries. We measured this tax: in our experiments, Focus agents generated on average 11,556 output tokens (for Haiku) compared to 18,158 for baseline agents. The \textit{additional} tokens for summary generation are offset many times over by the input token savings from not re-processing history.

For short, trivial tasks, this tax may outweigh the benefits. However, for long-horizon exploration (40+ steps), the tax becomes negligible compared to the cost of carrying a massive history. It effectively amortizes the cost of ``memory'' over the lifetime of the agent.

\subsection{Limitations}
Several limitations warrant discussion:

\begin{enumerate}
    \item \textbf{Sample Size:} Our evaluation uses N=10 (Haiku) and N=3 (Sonnet) instances. While sufficient to demonstrate both efficiency and reasoning benefits, the high computational cost of evaluating long-horizon agentic workflows limited our current sample size. How these results generalize to the full benchmark over longer evaluation runs remains to be seen.
    
    \item \textbf{Heuristic Triggers:} The current implementation uses a time-based ``nudge'' (every 15 steps) to trigger compression. This simple heuristic works but may be suboptimal. Future work could explore adaptive triggers based on information density or task phase detection.
    
    \item \textbf{Information Loss:} Compression necessarily discards details. While our knowledge preservation mechanism retains key learnings, edge cases may exist where compressed information was actually needed. We did not observe this in our experiments, but it remains a theoretical concern.
    
    \item \textbf{Programmatic Triggers:} Our implementation programmatically triggers focus completion rather than relying on the agent to self-invoke compression. This external orchestration may be necessary for reliable memory management.
\end{enumerate}

\section{Conclusion}
We have demonstrated that active context management provides two distinct benefits for LLM agents:

\begin{enumerate}
    \item \textbf{Cost Efficiency:} For capable models (Sonnet), Focus reduces token consumption by 51\% while maintaining 100\% task success---a direct cost savings with no capability loss.
    
    \item \textbf{Improved Reasoning:} For smaller models (Haiku), Focus improves success rate by 50\% through cognitive hygiene. By keeping context clean and focused, smaller models can reason more effectively without distraction from accumulated noise.
\end{enumerate}

The key insight is that active context compression converts the hard limit of context windows into a sustainable ``cognitive tax.'' By preserving learnings in a structured knowledge block while pruning raw interaction history, agents can explore more deeply and reason more clearly.

This finding has practical implications: organizations can potentially use smaller, cheaper models with Focus to achieve results comparable to larger models, or use Focus with larger models to dramatically reduce costs. The Focus architecture provides a practical, complementary approach to existing memory management techniques, operating entirely within the context window without requiring external infrastructure.

Future work will explore adaptive compression triggers, hierarchical knowledge structures, and the relationship between model capability and optimal compression frequency.
